## 🧭 ГЛАВНЫЙ ПРИНЦИП (повторён Владимиром дважды)
**Движок НЕ выдумывает данные, которые у нас УЖЕ есть.** Frozen/Dry, габариты, вес —
берём из НАШЕЙ базы (`SkuShippingData`, источник = теги Veeqo + прошлый опыт; новое вносим
вручную, и оно запоминается). Мозг их НЕ угадывает. (Совпадает с правилом CLAUDE.md: «не
угадывать тип товара — только по тегам Veeqo».) Зона мозга = опознать ПРОДУКТ (бренд, линейка,
вкус, контейнер, кол-во единиц, разбор вариаций) + где купить + цена + контент/фото.

## 🌐 УНИВЕРСАЛЬНЫЙ ДВИЖОК — 2 источника данных (уточнение Владимира 2026-06-08)
Движок один для нескольких модулей, и у него ДВА режима источника:
1. **Опознать НАШ текущий товар** (обогащение существующей базы): вход = наш листинг
   (title + при необходимости Veeqo-картинка как подспорье). Veeqo-фото — ТОЛЬКО для этого.
2. **Собрать контент** (для COGS + СОЗДАНИЯ НОВЫХ листингов/бандлов/сайта): описание, bullets,
   **≥5 фото (incl nutrition label)**, **Nutrition Facts**, **состав/ingredients** — берём из
   **ПЛАТНОГО сервиса** (карточка ритейлера first-party), НЕ из Walmart Seller Central.

**БД должна быть удобной** — по товару легко вытащить полный набор (description, все картинки,
bullets, ключевые моменты, Nutrition Facts, состав). Перспектива — **собственный сайт**: агент
создаёт карточки прямо из базы. ⇒ контент храним полно (RetailPrice уже имеет
title/description/keyFeatures/imageUrls — дополнить nutritionFacts + ingredients + ≥5 imageUrls).

## Cквозное требование — ГДЕ ЛУЧШЕ КУПИТЬ (5 площадок)
Движок должен ценить каждый продукт минимум на **5 ритейлерах: Walmart, Target, BJ's, Sam's Club, Publix** (Владимир сам так закупается каждый день — заказывает там, где выгоднее). Хранить цену по каждому + **лучший источник (где купить)**. (Упирается в платный тариф мульти-ритейлера — развилка по бюджету.)

## ▶️ Приоритет фиксов
1. **Не выдумывать** category/dims/weight → брать из `SkuShippingData` (быстро, принципиально).
2. **is_bundle** починить определение (быстро).
3. **Жёсткий first-party фильтр** в Stage B + чинит битые матчи (La Abuela).
4. **Картинки:** дотянуть титульное фото (Veeqo) → мозг видит фото → разбор вариаций (Green Giant ×4).
5. **Полный сбор** с карточки: описание + bullets + атрибуты + ≥5 фото.
6. **Переделать COGS-вкладку** (понятный формат) + вкладка вариаций с компонентами.
7. **Мульти-ритейлер 5 площадок** + лучший источник (после решения по бюджету).
