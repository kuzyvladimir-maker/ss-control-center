# 🧠 Wiki-Brain — авто-санитар базы знаний

**Статус:** работает с 2026-06-20.
**Зачем:** держать «вики-мозг» (docs/wiki/ + память) актуальным и связным
**без ручной дисциплины** — система сама ловит «зарастание» и тычет носом.

---

## Проблема, которую решает

Раньше актуальность держалась на памяти человека (и Claude): не забыть дописать
статью, добавить ссылку, обновить оглавление. Это забывается → копятся:
- **сироты** — статьи, на которые никто не ссылается (их не найти из оглавления);
- **битые ссылки** — ссылка ведёт в несуществующий файл;
- **расхождение код↔доки** — код поменяли, а документацию к нему не тронули;
- **рассинхрон памяти** — заметка `memory/*.md` не попала в индекс `MEMORY.md`.

## Из чего состоит

**1. Движок-аудитор** — [`scripts/wiki-brain.mjs`](../../scripts/wiki-brain.mjs).
Чистый Node.js, без зависимостей. Строит граф связей по markdown-ссылкам и проверяет:

| Проверка | Что значит | Жёсткость |
|---|---|---|
| Сироты | вики-файл без входящих ссылок | жёсткая (валит аудит) |
| Битые ссылки | ссылка на несуществующий файл | жёсткая |
| Память не в индексе | `memory/*.md` нет в `MEMORY.md` | жёсткая |
| Тупики | файл сам ни на что не ссылается | мягкая (норма для листовых статей) |
| Дрейф код↔доки | менялся код в `src/`, но `docs/` не тронут | мягкая (напоминание) |

Файлы-хабы (`index.md`, `CONNECTIONS.md`, `SESSION-HANDOFF.md`) от сиротства освобождены.

**2. Хуки Claude Code** — [`.claude/settings.json`](../../.claude/settings.json).
Это и есть «дисциплина без ручного труда»: движок запускается сам.

| Хук | Когда | Что делает |
|---|---|---|
| `SessionStart` | старт сессии | печатает строку статуса мозга |
| `PostToolUse` (Write/Edit) | сразу после правки вики/памяти | блокирует с подсказкой, если создал сироту или битую ссылку |
| `Stop` | перед завершением ответа | проверяет незакоммиченные изменения (дрейф/новые сироты); блокирует один раз |

## Как пользоваться вручную

```bash
node scripts/wiki-brain.mjs            # полный аудит, человекочитаемо
node scripts/wiki-brain.mjs --json     # машинный вывод
node scripts/wiki-brain.mjs --graph    # + пересобрать docs/wiki/.brain-graph.json (граф связей, god-nodes)
```

Выход: код `0` — чисто, `1` — есть жёсткие проблемы.

## Чего система НЕ делает

Она **не пишет текст статей сама** — это требует интеллекта (делает Claude). Её
работа — поймать проблему и заставить починить. То есть граница такая:
обнаружение и принуждение к дисциплине — автоматические; наполнение смыслом — за Claude.
