# Listing Quality Stack — общий фундамент для создания И улучшения листингов

> **Owner-принцип (Vladimir, 2026-06-27):** логика «как делать хороший листинг»
> (картинки, полный набор атрибутов, сборка по базе знаний, проверка QA-офицером)
> — это **один общий стек**, а не три копии. Им пользуются и фабрика новых
> листингов, и улучшатели текущих.

## ⚡ Суть

Один и тот же набор знаний и модулей обслуживает **три** продукта:
- **[Bundle Factory](bundle-factory.md)** — СОЗДАНИЕ новых листингов.
- **[Amazon Growth](amazon-growth-roadmap.md)** — УЛУЧШЕНИЕ существующих листингов Amazon.
- **[Walmart Growth](walmart-growth-roadmap.md)** — УЛУЧШЕНИЕ существующих листингов Walmart.

Создание и улучшение — это одна и та же задача «довести листинг до эталона»,
только с разных стартовых точек (с нуля vs с текущего состояния). Значит и эталон,
и инструменты должны быть общими.

## 🧱 Из чего состоит стек

1. **Marketplace KB (база знаний)** — единый источник правды per-канал:
   - Полный перечень **атрибутов** per product type, выкачанный из API самих
     маркетплейсов (Amazon `getDefinitionsProductType`: GROCERY 98, FOOD 117,
     GOURMET_FOOD 81, CHOCOLATE_CANDY 110, COFFEE/TEA 113 и т.д.; Walmart MP_ITEM
     Food & Beverage). Лежит в `docs/marketplace-rules/{amazon,walmart}/_schemas/`.
   - **Правила картинок** (main vs secondary, размеры, что запрещено).
   - **Правила контента/бренд-войс** (title/bullets/description, без эмодзи/промо).
   - **Важно:** найдено, что у Amazon НЕТ типа `GIFT_BASKET` — реальный тип
     GROCERY/FOOD/GOURMET_FOOD, «gift basket» = browse-node-позиционирование.
   - Сейчас KB читает только Bundle Factory (через `kb-loader`). Цель — чтобы
     его читали и оба Growth-модуля.

2. **Картинки** — общий принцип «откуда брать и как генерить»:
   - Вторичные фото = **реальные фото из донорского каталога** (`DonorProduct.imageUrls`).
   - Главная (титульная) = **пересобирается** по шаблону. Для frozen — наш
     брендированный кулер + гелевые пакеты «FROZEN GEL PACK» + товар внутри
     (это и есть товар лицом — Salutem-брендированный gift set). Спека:
     `docs/BUNDLE_FACTORY_FROZEN_MAIN_IMAGE_v1.0.md`, эталоны в
     `public/bundle-factory/frozen-refs/`.
   - Генерация — **бесплатный Codex/ChatGPT-subscription воркер** (gpt-image-2,
     $0), никогда платный OpenAI Images API. Воркер дорастить под референс-картинки.
   - Применяется для всех каналов **кроме Walmart** (Walmart запрещает frozen).

3. **Полнота атрибутов** — алгоритм проходит по всему перечню из KB и по каждому
   полю решает: заполнить (из данных каталога/KB), пропустить или «нужны данные».
   Это одинаково нужно и при создании, и при аудите (Amazon Growth уже считает
   «content completeness», но без полного списка из KB).

4. **Brand voice** — единый набор правил (без эмодзи/промо/health-claims,
   дисклеймер куратора). СЕЙЧАС продублирован в 3 местах
   (`compliance/banned-words.ts`, `walmart/multipack/content.ts`, advisor) —
   подлежит вынесению в общий модуль.

5. **Qualification Officer (QA-офицер, Отд 5 «Квалификация»)** — умный
   pre-publish/pre-update гейт: проверяет КАЖДЫЙ листинг на соответствие KB
   маркетплейса (правила + полнота атрибутов + картинки + бренд-войс + IP) перед
   выпуском. Тот же офицер обслуживает и публикацию новых, и применение улучшений.

## 🚦 Текущее состояние (по аудиту 2026-06-27)

- **Общее уже сейчас:** Marketplace KB (но читает только Bundle Factory),
  Compliance Gate (BF + частично Amazon Growth), SP-API/Walmart клиенты, Prisma.
- **Дублируется / НЕ общее:** brand-voice (×3 копии), валидаторы контента,
  учёт атрибутов (несовместимые схемы), генерация картинок (только BF),
  QA-офицер (ещё нет).
- **Growth-модули сегодня НЕ читают KB и не генерят контент/картинки** — они
  диагностируют (Amazon — свой Listing Health; Walmart — нативный LQ score).

## 🎯 Целевой принцип

Один KB + **общие модули** (`brand-voice`, обобщённый `marketplace-kb` loader,
реестр атрибутов, сборщик картинок, QA-офицер), которые **импортируют** все три
продукта — а не копии в каждом. Извлечение в общие модули делается поэтапно во
время пересборки Bundle Factory, и затем Growth-модули переключаются на них.
