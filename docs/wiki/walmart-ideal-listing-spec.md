# 🎯 Эталон идеального листинга Walmart (qualification target)

> **Зачем:** единый «идеальный образ» листинга, с которым движок квалификации
> СРАВНИВАЕТ каждый наш листинг. Не абстрактно «улучшить», а «дотянуть до вот
> ЭТОГО». Собрано из официального гайда Walmart (Marketplace Learn) —
> [[content-and-listing-quality]], [[images]] (`docs/marketplace-rules/walmart/`),
> плюс brand-voice владельца (CLAUDE.md) и логика мультипаков.
> Связано: [[walmart-growth-listing-quality]], [[title-policy]],
> [[multipack-policy]], [[mp-item-food-attributes]]. Память:
> `project_walmart_remediation_worker`, `project_bluecart_dropped_unwrangle_walmart`.
>
> **Создано:** 2026-07-02. Числа — из гайда Walmart; перепроверять перед прогоном.

Листинг «идеален», когда ВСЕ 6 блоков ниже = зелёные. Движок квалификации ставит
каждому листингу оценку по этим пунктам (identity-gated: сначала «тот ли это товар»,
потом «дотянут ли до эталона»). Это заменяет прежний ложный критерий «есть main = A-до-Я».

---

## 0. Главный принцип — DISCOVERABILITY (весь текст+атрибуты работают на поиск)

Весь текстовый и структурный контент строится **под алгоритм поиска**, чтобы Walmart
(и AI-поиск) максимально точно понимал товар. Это главный рычаг ротации/индексации.

- **Полнота поисковых терминов, БЕЗ повторов.** Покрыть все релевантные слова, которыми
  реально ищут покупатели (тип, вкус, форма, повод, диета, размеры, синонимы), распределив
  их по заголовку / описанию / буллетам / **атрибутам**. Walmart **штрафует повтор** одного
  слова — поэтому не «набивать», а РАСШИРЯТЬ охват через разные поля и атрибуты.
- **Атрибуты = скрытый SEO.** Каждый заполненный атрибут = фильтр/фасет/поисковый сигнал
  (см. блок 6). Чем больше корректно заполнено — тем в большем числе поисков и фильтров
  листинг всплывает. Это, по сути, основной способ «легально» расширить охват ключевых слов.
- **Адаптация под LLM/AI-поиск (GEO/AEO).** Всё больше людей ищут товар через языковые модели.
  LLM лучше «понимает» листинг, когда текст: (а) фактический и однозначный (что это, из чего,
  сколько единиц, как хранить, для чего), (б) с явными сущностями/атрибутами (бренд, вкус, вес,
  количество — прямым текстом, а не намёком), (в) естественным языком, отвечающим на типовые
  вопросы покупателя. Наш factual brand-voice (без промо/emoji) как раз этому и способствует.
- **Точность обязательна** (правило Walmart): все слова/картинки должны совпадать с реальным
  товаром. SEO не в ущерб правде — иначе content-issue и падение LQ.

## 1. Главная картинка (MAIN) — жёстко

| Критерий | Эталон | Как проверяем |
|---|---|---|
| Тот товар | бренд + тип + вкус/вариант с этикетки == заголовок листинга | vision `frontMatchesListing` (fail-closed) |
| Количество | показано ровно **N единиц** (N = сколько реально в паке) | vision-verify считает единицы |
| Фон | seamless white **RGB 255/255/255** | детерминированный тайл на чистом белом |
| Заполнение | **максимум (~95%)** — используем пространство по максимуму | ОСОЗНАННО оставляем 95% (решение владельца): почти всё — мультипаки, при 8/9/12 единицах каждая уже мелкая → надо крупно, чтобы товар был узнаваем даже в поисковой миниатюре. Walmart-овские 75-90% — мягкая рекомендация, не hard-reject → 95% безопасно |
| Разрешение | рекоменд. **2200×2200**, минимум для zoom 1500×1500 | ⚠️ сейчас 2000×2000 → поднять до 2200 (крупнее единицы при zoom) |
| Формат/размер | JPEG/PNG, 1:1, RGB, ≤5 MB | ok (PNG ~2 MB) |
| Запрещено | текст/баннеры/водяные знаки/логотип продавца/чужие предметы/стоковое фото/лайфстайл | vision-гейт отбивает |

## 2. Вторичные картинки (secondary) — до 8

- **≥4 картинки всего** на листинг (рекомендация Walmart).
- Для grocery: **панель Nutrition Facts** отдельной вторичной картинкой (сильно рекомендуется).
- Плюс: реальные фото товара с других ракурсов, ingredients-панель, инфографика с количеством («1 заказ = N штук») — инфографика/текст на ВТОРИЧНЫХ разрешены.

## 3. Заголовок (title)

- **≤150 символов**, Title Case (не ALL CAPS).
- Структура: **Brand + ключевые атрибуты (размер/вес/count) + название товара**.
- Есть маркер количества: «N-Pack» / «N Count».
- **Запрещено:** промо-слова, спецсимволы, URL, имена ритейлеров/конкурентов, повтор слов, годы, не-английский (кроме имени бренда).
- Brand-voice владельца: без emoji, без promo-прилагательных (ultimate/best/premium…).

## 4. Описание (description)

- **≥150 слов**, прозой (НЕ буллеты), фичи+бенефиты, бренд + ключевые слова естественно.
- Ведущее предложение про количество: «This listing is a multipack of N …: one order ships all N together — selecting quantity 1 sends N, not 1.»
- Curator/assembler disclaimer для gift-basket линеек (см. CLAUDE.md).
- **Запрещено:** emoji, промо, буллеты, URL, спецсимволы, повтор слов, HTML для grocery.

## 5. Ключевые особенности (key features / bullets)

- **3-10 буллетов**, каждый **≤80 символов** (с пробелами).
- Короткие фактические фразы, без повторов, без emoji/HTML/промо.

## 6. Атрибуты (Walmart MP_ITEM 5.0) — МАКСИМАЛЬНАЯ полнота = главный SEO-рычаг

Правило владельца: **чем больше корректно заполненных атрибутов, тем лучше** (индексация,
фильтры, поиск, LLM). Заполняем ВСЁ применимое, не только минимум из 6.

- **Quantity trio (обязательно):** `multipackQuantity`=N / `countPerPack`=1 / `count`=N.
- **SAFE free-text (заполнять всегда, из донора):** `manufacturer`, `ingredients`,
  `foodAllergenStatements`, `flavor`, `flavor_notes`, `netContentStatement`, `cuisine`,
  `productLine`, `texture`, `occasion`, `vegetable_type`, `fruitType`, `dietaryMethod`,
  `manufacturerPartNumber`, `size`.
- **Closed-list enum (заполнять; значения донора Walmart-валидны, на reject само-снимается):**
  `containerType`, `container_material`, `foodForm`, `food_condition`, `spiceLevel`,
  `sizeDescriptor`, `food_preparation_method`, `ib_retail_packaging`, `productNetContentUnit`+`productNetContentMeasure`.
- **Ещё НЕ заполняем, а НАДО (todo движка, поднимет полноту):** `nutritionFactsLabel`
  (URL панели Nutrition Facts из донор-галереи — обычно она уже есть), `ingredientListImage`,
  вложенные `nutrientName/nutrientAmount/nutrientPercentageDailyValue`, `assembledProductWeight`.
  (`pieceCount` — только для наборов РАЗНЫХ товаров, не для наших одинаковых мультипаков.)
- Правильный **Product Type** выбирает применимый набор required-атрибутов + style guide.
  `brand` НЕ шлём для чужих каталог-карточек (QARTH `ERR_EXT_DATA_0101119`).
- Механизм: `buildFoodAttributes()` уже отдаёт SAFE+closed; feed-reject называет поле в
  `checkFeed.detail` → снять его, остальное оставить. Спека: [[mp-item-food-attributes]].

## Как это использует движок (Movement #1 — квалификация + улучшение)

```
1. Читаем НАШ справочный каталог (RetailPrice/DonorProduct) — весь текст + все фото по товару.
2. Нет данных / не хватает до эталона → зовём движок обогащения:
     • Oxylabs  = прямой walmart.com 1P: чистое главное фото + title + price (+ product detail).
     • Unwrangle = Target/Sam's/Costco: доп. контент/буллеты/доп. фото (ingredients, nutrition).
   Всё добытое ЗАПИСЫВАЕМ обратно в каталог (не тянуть повторно).
3. Квалификация: сравниваем листинг с ЭТАЛОНОМ (блоки 1-6), identity-first.
     • score по каждому блоку + список чего не хватает.
     • «A-до-Я ✓» ТОЛЬКО когда блоки 1-6 зелёные (а не «есть картинка»).
4. Улучшаем недотянутое (фото пере-собираем, текст/атрибуты дописываем), снова квалифицируем.
5. На проверку владельцу (галерея до/после) → публикация.
```

**Координация с чатом COGS/каталога:** тот же справочный каталог и те же платные
сервисы. Там: Oxylabs=цена, Unwrangle=контент. Здесь добавляем: Oxylabs ещё и
**чистое 1P-фото Walmart** (проверено — 5-7с, структурно). Один каталог кормит оба.
