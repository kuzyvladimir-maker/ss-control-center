## 1. Ценообразование

**Проблема (прогон 2026-07-01):** цены на 3 живых Uncrustables вышли **слишком высокие** — надо исправить.

**Вход оператора (на экране создания пула листингов):** блок-калькулятор в стиле ChannelMax (скриншот давал владелец; калькулятор уже частично вшит — `computeBundlePrice` / `PricingModal`). Оператор задаёт **либо целевую маржу, либо целевой ROI** — алгоритм считает цену сам.

**Себестоимость (COGS) — ТОЛЬКО из справочного каталога** (`DonorProduct` / Reference Catalog), не выдумывается.

**Разные алгоритмы ценообразования по категории (ключевое правило):**
- **Frozen (заморозка)** → алгоритм с кулером: COGS + пенопластовый кулер (S/M/L/XL по вместимости) + гелевый лёд + доставка frozen + referral + целевая маржа/ROI.
- **Refrigerated (охлаждёнка)** → тоже требует кулер (уточнить у владельца, но по логике cold-chain — как frozen).
- **Dry / shelf-stable (не заморозка и не охлаждёнка)** → БЕЗ кулера/льда: обычная коробка + доставка ambient + referral + маржа/ROI. Отдельный алгоритм.

## 2. Титульные картинки

**Порядок слотов:** сгенерированная **инфо-карточка о компании** (кто мы, наши принципы, почему цена выше / cold-chain) идёт **позицией #1 сразу после главной картинки** (не в конце галереи). **Генерация:** GPT Image 2.0 + **2 референса**: (1) наш брендированный кулер с нашим брендированным гелевым льдом; (2) товар «лицом».

### 2a. Uncrustables (бренд-исключение) — MAIN картинка

**Баг:** MAIN всегда генерится одинаковой — 4 коробки × 4 штуки = 16, независимо от тайтла (30/45/90). Картинка должна **соответствовать количеству** в тайтле.

**Реальные розничные фасовки Uncrustables (которые мы используем):** **4, 10, 15 штук в коробке.**

Два варианта, как чинить (РЕШЕНИЕ владельца — pending):
- **Вариант 1 — правдивые коробки, суммирующиеся в count.** 30 шт → 3 коробки × 10, ИЛИ 2 коробки × 15. Алгоритм подбирает реальные фасовки (4/10/15) так, чтобы сумма = количеству в тайтле.
- **Вариант 2 — индивидуальные Uncrustables в поштучной упаковке.** Каждый Uncrustable в индивидуальной полиэтиленовой упаковке, **цвет по вкусу** (30 клубничных → красноватый оттенок, как в оригинале). Красивее. НЕ показывать голый Uncrustable без упаковки (глупо смотрится).

### 2b. Gift-set (не-Uncrustables, frozen/chilled для Amazon)

- На титульной картинке писать **«Gift Set»** — на самом кулере (рядом с нашим лого) + на коробках вложенных продуктов.
- Композиция набора — см. раздел 3 (пример Jimmy Dean).

## 6. Диагностика цены 2026-07-01 — почему $144.84 (30-ct) слишком дорого

Реальные данные из прод-базы (SKU `AZ-ASMY-VEQ2`, ASIN B0H788M8WM):
`cost_breakdown = {goods_cents:2970, packaging_cents:1072}`, price $144.84, category FROZEN_GROCERY.

Точный разбор текущей формулы:
1. Товар (COGS): 30 × $0.99 = **$29.70**
2. Упаковка: **$10.72** (кулер **M** $9 + лёд $0.72 + коробка $1)
3. Наш лейбл доставки: **$32.00** (M) — **зашит в цену**
4. Итого затрат: **$72.42**
5. Цена = $72.42 ÷ (1 − 0.15 referral − 0.35 маржа) = ÷0.50 = **$144.84**
6. **+ клиент отдельно платит $22.07** доставку → клиент платит **$166.91**

**Два бага (сверх маржи 35%→30%):**
- 🔴 **Двойная доставка.** Лейбл зашит в цену товара И клиент платит доставку отдельно ($22.07 на скрине). Доставку платит клиент — её НЕ должно быть в цене товара. Фикс: `own_shipping_cents` НЕ включать в `total_cost` при solve цены (клиент платит через marketplace delivery). Затрагивает `computeBundlePrice` в `pricing-config.ts`.
- 🔴 **Кулер всегда M.** В пайплайн приходит `weight_lb = null` (есть `total_weight_oz`, но не конвертится в lb для цены), и код падает в дефолт «M cooler estimate». 30 Uncrustables (~3.75 lb товара) реально едут в **S**. Фикс: конвертить `total_weight_oz`→lb (или считать вес по товару) перед `computeBundlePrice`.

**Корректная цена (доставка вон + кулер S + маржа 30%):** товар $29.70 + упаковка S ~$7.30 = $37.00; цена = $37.00 ÷ 0.55 = **~$67.27** (клиенту ~$87 с доставкой ~$20 vs старые $166.91). ROI-70% → **~$74.00**.

**Статус:** маржа 30% + ROI-режим — сделано (коммит `f4faffc`). Доставка-вон-из-цены + фикс кулера — pending owner confirm (дефолт margin vs ROI; шиппинг в цене товара vs отдельной строкой).

## 7. ROI-формула — canonical (уточнено владельцем 2026-07-01)

`price = [ (COGS + packaging) × (1 + ROI) + shipping ] ÷ (1 − referral)`

- База ROI = COGS + упаковка (кулер+лёд+коробка). Прибыль = ROI × база (70% → ×1.70).
- **Компенсация fee:** делим на `(1 − referral)`, чтобы после referral остаться при целевой прибыли.
- **Компенсация shipping:** прибавляем стоимость лейбла (кулер-зависимую) — НЕ прибыль, а покрытие.

**Кулер для Uncrustables — по КОЛИЧЕСТВУ штук** (не по весу), справочник `src/lib/pricing/cost-model.ts` `coolerFor()`: **1–30 → S, 31–60 → M, 61–72 → L, 73–135 → XL**. Упаковка (`PACKAGING`): S $7.50 / M $10.90 / L $14.10 / XL $18.90. Лейбл (`LABEL`): S $20 / M $32 / L $45 / XL $60. Юнит-COGS Uncrustables = $1.

**Пример 30-ct:** база $30 + $7.50 = $37.50 → ×1.70 = $63.75 → +$20 shipping → ÷0.85 = **$98.53** (vs старые $144.84+$22.07=$166.91).

**РЕШЕНО (owner 2026-07-01, скриншоты бестселлеров + SP-API):** доставка идёт **ОТДЕЛЬНОЙ строкой** через Amazon shipping-template, **НЕ в цене товара**. Цена товара привязывается к реальным бестселлерам.

## 9. PRICING MODEL — привязка к бестселлерам (canonical, заменяет ROI-формулу для Uncrustables)

Цена товара (БЕЗ доставки) = **(COGS + packaging) × MARKUP**, где MARKUP ≈ **2.3–2.4** (усреднено с бестселлеров: 24шт 2.45–2.55× · 30шт 2.30× · 48шт 2.31×). Доставка — отдельно через template.

Проверка (markup 2.35, кулер по количеству, доставка отдельно):
- 30 шт: база $37.50 × 2.35 = **$88.13** + template-доставка (vs бестселлер $86.15 ✓)
- 45 шт (M): база $55.90 × 2.35 = **$131.37**
- 48 шт (M): база $58.90 × 2.35 = **$138.42** (vs бестселлер $135.99 ✓)
- 90 шт (XL): база $108.90 × 2.35 = **$255.92**

Эффективная маржа ~30–35% all-in (net-ROI ~70% если в базу включить и наш лейбл ~$20). **Реализация:** markup-режим ~2.35 на (COGS+упаковка), доставка ВОН из цены, кулер по количеству, прикрепить frozen-template GUID + вес по бэнду.
