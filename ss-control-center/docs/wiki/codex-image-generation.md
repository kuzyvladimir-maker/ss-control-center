# Codex Image Generation (FREE ChatGPT-subscription path)

**Status:** Live — 2026-06-25
**Replaces:** paid OpenAI Images API (`openai.images.generate`, gpt-image-1/2)

## TL;DR

Listing & A+ images are now generated by the **Codex CLI's built-in `image_gen`
tool** (model `gpt-image-2`) running on a **ChatGPT subscription** — **$0 per
image**. The paid OpenAI Images API path has been removed entirely. OpenAI API
spend on `images` should be **0** going forward.

Text/vision OpenAI usage (customer-hub analyzers, `ai-vision.ts`) is unchanged —
this only swapped the *image* provider.

## Why a remote worker (and not just call Codex from the app)

SS Command Center runs on **Vercel serverless**. The Codex CLI is a stateful
local tool: it needs `~/.codex/auth.json` (the ChatGPT subscription session) and
a writable `~/.codex/generated_images/` dir. Neither exists on Vercel.

So generation lives on the **OpenClaw box** (`104.219.53.204`, always-on Linux),
which is already `codex login`-ed with a ChatGPT subscription. A tiny HTTP service
wraps `codex exec` and the Vercel app calls it instead of OpenAI.

```
Vercel (generateMainImage)
   │  POST { prompt, size }  + Bearer token
   ▼
https://mcp.salutem.solutions/codex-image/generate   (nginx :443, Let's Encrypt)
   │  proxy_pass
   ▼
127.0.0.1:8791  codex-image-worker.service (systemd, Node)
   │  spawn: codex exec --skip-git-repo-check "...Use the imagegen skill..."
   │         (OPENAI_API_KEY / CODEX_API_KEY stripped from env)
   ▼
Codex built-in image_gen (gpt-image-2, ChatGPT subscription)
   │  writes ~/.codex/generated_images/<session>/ig_*.png
   ▼
worker returns image/png bytes  ──►  Vercel uploads to R2  ──►  permanent URL
```

## The free path vs the paid path (critical)

The Codex `imagegen` skill (`~/.codex/skills/.system/imagegen/`) has **two** modes:

| Mode | Auth | Cost | Used? |
|------|------|------|-------|
| Built-in `image_gen` tool | ChatGPT subscription | **$0** | ✅ this is what we use |
| CLI fallback `scripts/image_gen.py` | needs `OPENAI_API_KEY` | **pay-per-image** | ❌ never |

The worker strips `OPENAI_API_KEY` and `CODEX_API_KEY` from the child env (and the
systemd unit sets them empty), so the paid `scripts/image_gen.py` fallback *cannot*
run even by accident. The only path available is the subscription built-in tool.

## Code

- **`src/lib/image-gen/codex-worker.ts`** — `generateImagePngViaCodex({prompt,size})`.
  POSTs to the worker, returns PNG bytes. Normalizes output to the requested size
  with `sharp` (the built-in tool returns its own resolution, ≈1254² square, and
  ignores exact pixel requests — `fit: cover` fixes aspect ratio for A+ landscape
  vs square main images). Test/smoke hook: `globalThis.__SS_CODEX_IMAGE_STUB__`.
- **`src/lib/bundle-factory/image-generation.ts`** — `generateMainImage()` unchanged
  signature; internals now call the worker, upload to R2, and return a permanent
  URL. `cost_cents` is now **always 0** (kept for DB compat). Dev-mock placeholder
  when the worker env is unset.
- **`src/lib/amazon/aplus/images.ts`** — unchanged call site; the `model`/`quality`
  params are now no-ops (subscription path picks the model). `size` still honoured.

### Env vars (local `.env.local` + Vercel Production)

```
CODEX_IMAGE_WORKER_URL=https://mcp.salutem.solutions/codex-image/generate
CODEX_IMAGE_WORKER_TOKEN=<shared secret, matches the box .env>
```

Without these (and no stub), `generateMainImage` returns a placehold.co
placeholder (`mock_mode: true`) so the pipeline/UI still work in dev.

## The worker on the box

- Code: `/root/codex-image-worker/server.js` (plain Node, no deps)
- Secret + config: `/root/codex-image-worker/.env` (chmod 600)
- Service: `systemctl {status,restart,stop} codex-image-worker`
- Logs: `journalctl -u codex-image-worker -f`
- nginx: `location /codex-image/` block in
  `/etc/nginx/sites-available/mcp.salutem.solutions` (300s proxy timeout)
- Endpoints: `GET /health` → `{ok:true}`, `POST /generate` (Bearer) → `image/png`
- Requests are **serialized** (one `codex exec` at a time) — generation writes to a
  shared dir, so serializing keeps output detection unambiguous and is gentle on
  subscription rate limits. Image gen is operator-driven (low volume), so fine.
- After returning a PNG the worker deletes that `generated_images` session dir.

## Operating notes / gotchas

- **Latency:** ~30–90s per image (subscription agent run), vs ~10s for the old
  OpenAI API. The Vercel image routes have `maxDuration=300` and retry up to 3×
  per channel — a multi-channel draft with retries can get close to that ceiling.
  If it becomes a problem, reduce retries or move bulk generation to an async job.
- **Output size:** built-in tool returns ≈1254² and is normalized down/up to the
  requested `size`. Note `validator-image-dimensions` wants Amazon ≥2000² /
  Walmart ≥1500²; the current request sizes (1024²/1536×1024) are under that —
  this was already true with the old 1024² OpenAI path. To satisfy it, bump the
  caller `size` (the subscription tool + sharp upscale support larger).
- **Subscription session:** `codex login` tokens auto-refresh. If image gen starts
  failing with 502s, check `ssh openclaw 'codex login status'` — it must say
  "Logged in using ChatGPT" (NOT "API key"). Re-login with
  `codex login --device-auth` if the refresh token was revoked.
- **Box must stay up.** If `104.219.53.204` / nginx / the service is down, image
  gen returns errors (fail-closed) — it never silently falls back to a paid path.
- **Verify $0 spend:** OpenAI Platform → Usage → Images should stay flat. The code
  has no `images.generate` call anymore, so spend is structurally 0 regardless.
